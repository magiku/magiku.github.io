技术栈

以 GitHub Page + Hexo 为脚手架，再集成各类现成的轮子，辅助修改而成。

网上有很多教程与插件，甚至各种复读贴，所以就不多介绍了。

搭建原因

最近觉得需要自己做技术笔记和过往故障处理案例，不然容易忘了且不好捡回来。

体验了很多软件跟网站后，决定还是使用本地编写+云端共享的方式。

回家写笔记，平时手机查阅，这样就合适。

好吧，还是因为抠。这些厂商的服务一个个都要交钱才有好的体验。

掏钱是不可能掏钱的，而微软又经常网络抽风，这才自己搭建博客。

用现成的东西搭建是真的方便快捷，以前自己写要一个月。

网站情况

目前不确定会不会持续更新，因为在此之前我也没有做笔记的习惯。

不过我会试着写一写教程向的文章，不会像其他博客特别是CSDN那样的复制粘贴的。

另外，我考虑记录一些我看到好贴或者好视频，会将他们记录下来，相当于分享与存档吧。

我始终认为想把知识融会贯通，学习之后以教人的方式再整理复述出来才是最有效的。

我经常在教别人时才发现自己本以为学会的知识其实还有弄不清楚的地方。

关于转载

我本人秉承着知识不问出处的理念。我从来不关注一个方法论/知识点是哪本书的内容，是哪位学者的观点。

我赞同了、理解了并内化了才是最重要的，而不是像背书一样去记忆。（内化是指成为你自己的思维，形成自然。）

比如我上一段所讲的学习方法，好像被叫做“费曼学习法”，老实说，我并没看过，我就是自己一直这么做而已。

有的人总是想着：哦，我要参照“费曼学习法”。哦，我要使用“番茄时钟”来专注学习。哦，我要依据“马斯洛需求层次”来控制我自己。

我不确定他们是不是都做到了，但如果是我，这种遵照别人的规则来生活是坚持不下来的；只有内化为自己的东西，才能自然的使用。

说的有点远了，总之就是基于这样的理念，我不反对所有人对我网站内容的摘抄。

但希望你是为了学进去，如果只是复制粘贴发布，假装就是自己的东西，那对不起的只会是自己的时间。

叠甲：我并不是支持盗版，版权是商业化的东西，而我说的知识都是公开的东西。同时我的观点是知识内化后，不用去记忆从哪学来的，内化之后就是自己的东西了。（我要强调一点就是内化的东西是要自己理解后的东西，而不是照搬别人脑子里的东西。）

个人说明

个人简介

广东汕头人，在广州读的本科，读的是轮机工程，一个就业方向是出海跑船当维修工的专业。

农村出身的我没有人指点过专业志愿，当初只想着来大城市–广州，确定学校后，就按着学校对应的专业排名填志愿。（选的第二个专业是航海技术orz）

大二体验过海员生活后，确实接受不了船舶机舱的环境，就决心要转行。（死宅怎么可能受得了没网的地方，而且机舱又热又吵）

当时网上公认的唯二出路就是考公与编程，我本身就对计算机感兴趣，所以选择了后者。

不过由于毕业当年就遇上疫情，大多数公司都停招/缩招，毕业时没能找到开发岗位，也是脸皮薄不敢吹。

最后是脱下长衫，去了亲戚开的工厂当个月休2天的设备技术员（纯牛马），期间就开始学习运维方面的知识。

因为我清楚开发这一块已经在走下坡路，且对于非科班的人来说，毕业时没入行，那后面就几乎不可能再入了。

从那开始就在运维这个方向深入发展了。

网名的由来 | 点击展开

早年喜欢初音未来（miku），同时还喜欢音游deemo，尤其是里面的一首歌，彩华（saika）因此用过一段时间的彩华米库这个id。
另外还有一首歌也很喜欢，尤其是歌名MagiCatz的意译+音译，便取了“魔剂喵”这个id。目前B站跟一些游戏还是用的这个名字。
现在，上了年纪，不是mmr了，id带个喵还有点驾驭不住。（忘了在哪个平台魔剂猫有重名的，所以才一直用魔剂喵）
就给自己起了个洋名magiku————miku与MagiCatz各取一半，同时谐音magic。
当然，英文名在国际网站上难免遇到id重复的，所以我有时会加后缀030，除了是颜文字外，还是我的手机尾号。
目前网站名的[药剂屋]则是从“魔剂”两字衍生出来的。希望你能从这里得到好的“疗效”！

希望年后能找到好工作，也奉劝各位千万不要答应去接手某个在建项目（除非给你实权，或者有明确的补贴）。
